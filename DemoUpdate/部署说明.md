# 升级服务部署说明

## 文件清单

| 文件 | 说明 |
|------|------|
| `version.json` | 版本信息文件（必须） |
| `MediaNameFixer_0.1.0_x64-setup.exe` | 安装包（新版本发布时替换） |

## 部署步骤

### 1. 上传到服务器

将本目录下的所有文件上传到服务器目录：

```
https://techwebplus.cn/medianamefixer/
```

即最终可访问地址为：
- `https://techwebplus.cn/medianamefixer/version.json`
- `https://techwebplus.cn/medianamefixer/MediaNameFixer_0.1.0_x64-setup.exe`

### 2. 服务器配置要求

确保服务器支持 **CORS 跨域访问**，响应头需要包含：

```
Access-Control-Allow-Origin: *
```

### 3. 发布新版本流程

1. 打包新版本安装包（`npm run tauri build`）
2. 将生成的安装包复制到本目录
3. 修改 `version.json`：
   - `version`: 更新为新版本号（如 `0.1.1`）
   - `downloadUrl`: 更新为新的安装包下载地址
   - `releaseNotes`: 填写本次更新内容
4. 将更新后的文件重新上传到服务器

### version.json 格式说明

```json
{
  "version": "0.1.1",
  "downloadUrl": "https://techwebplus.cn/medianamefixer/MediaNameFixer_0.1.1_x64-setup.exe",
  "releaseNotes": "1. 修复了xxx问题\n2. 新增了xxx功能"
}
```

## 注意事项

- 版本号格式为 `x.y.z`，程序会自动对比大小
- `downloadUrl` 可以是任意有效下载链接
- 如果不想提供更新，保持 `version` 小于或等于客户端版本即可
